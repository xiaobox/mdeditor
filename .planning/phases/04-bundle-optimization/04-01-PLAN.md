Split the 4,788 kB monolithic production bundle into 6 domain-based vendor chunks using Vite's manualChunks function, while preserving Mermaid's existing ~27 dynamic import chunks.

Purpose: Cache-friendly vendor splitting allows browsers to cache stable vendor code independently from application code, reducing reload times after deployments.
Output: Modified vite.config.js with manualChunks function; build output containing 6 vendor chunks + preserved Mermaid dynamic chunks.
</objective>

    **Step 2: Add manualChunks function to vite.config.js.**
    Per D-01 (function form), D-02 (6 vendor groups), D-03 (ungrouped to default), D-04 (mermaid core only, not diagram sub-chunks):

    Define a `manualChunks` function BEFORE the `defineConfig` call. The function receives a resolved module `id` string and returns a chunk name or `undefined`:

    ```javascript
    /**
     * manualChunks - Split vendor dependencies into domain-based chunks
     * for cache-friendly production builds. (BLD-01)
     *
     * CRITICAL: Mermaid diagram sub-modules under /dist/chunks/ are
     * excluded to preserve their existing dynamic import splitting. (BLD-02)
     */
    function manualChunks(id) {
      if (!id.includes('node_modules')) {
        return undefined
      }

      // vendor-vue: Vue core + ecosystem
      if (
        id.includes('node_modules/vue/') ||
        id.includes('node_modules/@vue/') ||
        id.includes('node_modules/vue-i18n/')
      ) {
        return 'vendor-vue'
      }

      // vendor-codemirror: CodeMirror 6 editor + vue-codemirror wrapper
      if (
        id.includes('node_modules/codemirror/') ||
        id.includes('node_modules/@codemirror/') ||
        id.includes('node_modules/vue-codemirror/')
      ) {
        return 'vendor-codemirror'
      }

      // vendor-milkdown: Milkdown WYSIWYG + ProseMirror (includes @milkdown/prose)
      if (id.includes('node_modules/@milkdown/')) {
        return 'vendor-milkdown'
      }

      // vendor-mermaid: Mermaid core ONLY, NOT diagram sub-chunks (D-04)
      if (
        id.includes('node_modules/mermaid/') &&
        !id.includes('/dist/chunks/')
      ) {
        return 'vendor-mermaid'
      }

      // vendor-mathjax: MathJax rendering engine
      if (id.includes('node_modules/mathjax-full/')) {
        return 'vendor-mathjax'
      }

      // vendor-export: PDF/image export tools
      if (
        id.includes('node_modules/html2canvas/') ||
        id.includes('node_modules/jspdf/')
      ) {
        return 'vendor-export'
      }

      // All other node_modules: let Rollup decide (D-03)
      return undefined
    }
    ```

    Then add `rollupOptions` to the existing `build` section in `defineConfig`:

    ```javascript
    build: {
      outDir: 'dist',
      assetsDir: 'assets',
      rollupOptions: {
        output: {
          manualChunks
        }
      }
    }
    ```

    Preserve ALL existing configuration (base, plugins, server, resolve). Only add the `rollupOptions` block inside `build` and the `manualChunks` function definition above `defineConfig`.
